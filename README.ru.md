**Извлекайте AES-ключи контента, скачивайте и расшифровывайте зашифрованные
ресурс-паки Minecraft Bedrock из командной строки.** Кроссплатформенная CLI на
Go для исследователей безопасности, операторов серверов, проверяющих
собственные развёртывания, и авторов паков, восстанавливающих свои ключи -
снимайте ключи прямо с живого сервера Bedrock через Xbox Live, не имея ключа
заранее, а затем превращайте зашифрованные файлы `.mcpack` в обычные
редактируемые папки или зашифровывайте свои собственные.

- **`keys`** - вход через Xbox Live и извлечение AES-ключей контента сервера в `keys.json`, ключ заранее не нужен
- **`download`** - скачивание всех ресурс-паков, которые отдаёт сервер Bedrock, вместе с ключами, одной командой (добавьте `--decrypt` для готовых к использованию папок)
- **`decrypt`** - превращение зашифрованного `.mcpack` в обычный редактируемый каталог
- **`encrypt`** - упаковка обычного ресурс-пака в готовый к развёртыванию `.mcpack` + `.mcpack.key`
- **`featured`** - просмотр и скачивание из каталога Featured Servers / Live Events в Minecraft
- **`login`** - вход через Xbox Live (поток device code) и кэширование токена
- **`logout`** - удаление кэшированных файлов авторизации Xbox + MCToken
- **`version`** - вывод версии сборки
- **интерактивное меню** - запуск без команды открывает меню с разделами: просмотр Featured Servers (фильтрация по мере ввода, множественный выбор пробелом), ввод любого `host:port` и выбор действия (скачать, скачать + расшифровать или только ключи), расшифровка прошлых загрузок, шифрование локального пака или переход в Settings для входа/выхода и очистки кэшированных данных - и наблюдайте за прогрессом в реальном времени, ставьте на паузу клавишей `p`, отменяйте через `esc`

**Назначение.** Создано для исследователей, операторов серверов, проверяющих
собственные развёртывания, и авторов паков, восстанавливающих свои ключи. Не для
распространения чужого платного контента.

## Что это делает одной командой

```bash
# Скачать все зашифрованные паки, которые отдаёт сервер, затем расшифровать их за один шаг:
bedrock-pack-tools download --decrypt play.example.net:19132
```

Вы получаете обычные редактируемые папки паков плюс `keys.json` (UUID к AES-ключу) -
готовые к просмотру, сравнению или переупаковке. Запустите без аргументов, чтобы
вместо этого открыть интерактивное меню.

## Установка

### Готовый бинарник (быстрее всего, без инструментария Go)

Возьмите архив для вашей машины со
[страницы Releases](https://github.com/iteplenky/bedrock-pack-tools/releases/latest):

| Ваша машина | Архив |
|---|---|
| macOS, Apple Silicon (M1/M2/M3) | `bedrock-pack-tools_darwin_arm64.tar.gz` |
| macOS, Intel | `bedrock-pack-tools_darwin_amd64.tar.gz` |
| Linux, большинство ПК и серверов | `bedrock-pack-tools_linux_amd64.tar.gz` |
| Linux, ARM (например, Raspberry Pi 64-бит) | `bedrock-pack-tools_linux_arm64.tar.gz` |
| Windows, большинство ПК | `bedrock-pack-tools_windows_amd64.zip` |
| Windows, ARM | `bedrock-pack-tools_windows_arm64.zip` |

Скачайте и распакуйте (подставьте имя своего архива):

```bash
curl -L https://github.com/iteplenky/bedrock-pack-tools/releases/latest/download/bedrock-pack-tools_darwin_arm64.tar.gz | tar xz
./bedrock-pack-tools --help
```

Windows: распакуйте `.zip` и выполните `bedrock-pack-tools.exe --help`.

Каждый архив содержит один бинарник `bedrock-pack-tools`. Сверьте загрузку с `checksums.txt` со страницы Releases (`sha256sum -c checksums.txt`).

### Из исходников (Go 1.25+)

```bash
go install github.com/iteplenky/bedrock-pack-tools/v3@latest
```

Суффикс `/v3` обязателен. Бинарник попадёт в `$(go env GOPATH)/bin` - при необходимости добавьте этот путь в `PATH`.

### Вход через Xbox Live

Командам `keys`, `download` и `featured` нужен аккаунт Microsoft / Xbox Live.
При первом запуске вы увидите запрос device code:

```
Auth: no cached token - starting Xbox Live device auth
A URL and code will appear - enter it in your browser.
```

Токен кэшируется локально и переиспользуется при последующих запусках. Команда
`featured` также получает MCToken через PlayFab, который кэшируется отдельно
примерно на 4 часа.

Файлы кэша (`.xbox_token.json`, `.mctoken.json`, `.device_id`) находятся в
каталоге пользовательских настроек ОС:

- macOS: `~/Library/Application Support/bedrock-pack-tools/`
- Linux: `~/.config/bedrock-pack-tools/`
- Windows: `%AppData%\bedrock-pack-tools\`

## Быстрый старт

Запустите без аргументов, чтобы открыть интерактивное меню - выберите сервер из Featured или введите любой `host:port`, выберите действие (скачать, скачать + расшифровать или только ключи) и следите за прогрессом на месте (`p` ставит на паузу, `esc` отменяет):

```bash
./bedrock-pack-tools
```

В меню также есть **Decrypt packs**, **Encrypt a pack** и **Settings** (вход/выход, очистка сохранённых адресов или истории загрузок, сброс когорты featured).

Из командной строки:

```bash
./bedrock-pack-tools featured                                   # посмотреть, что доступно сейчас
./bedrock-pack-tools download --decrypt play.example.net:19132  # скачать + расшифровать за один шаг
```

## Команды

### `keys` - извлечение ключей шифрования сервера Bedrock

```bash
bedrock-pack-tools keys <server:port> [output.json]
```

Подключается к серверу Bedrock и записывает карту UUID → ключ. По умолчанию
вывод сохраняется в `<sanitized_server>_keys.json` (например, `play.example.net:19132`
превращается в `play_example_net_19132_keys.json`).

Используйте это, когда зашифрованные паки уже есть на диске и нужны только
ключи; в противном случае `download` делает и то, и другое сразу.

### `download` - скачивание ресурс-паков с сервера Bedrock

```bash
bedrock-pack-tools download [-v] [--decrypt] <server:port> [output-dir]
```

Скачивает все паки, которые отдаёт сервер, плюс файл ключей. Каждый пак
попадает в собственную папку `Name_vVersion/`. Обрабатывает как передачу паков
на уровне протокола, так и резервный вариант через CDN-URL, который используют
некоторые серверы.

`-d` / `--decrypt` расшифровывает паки сразу после скачивания, так что вы
получаете готовые к использованию папки за один шаг. Расшифрованные паки попадают
в `decrypted/<server>/` (сгруппированы по серверу, чтобы несколько загрузок не
смешивались); команда выводит точное расположение по завершении.

`-v` / `--verbose` выводит все ID пакетов для отладки проблем с handshake.

Без `--decrypt` скачанные паки остаются зашифрованными; инструмент выводит точную
команду `decrypt --all`, которую нужно запустить дальше.

### `decrypt` - расшифровка зашифрованных ресурс-паков

```bash
# Один пак с известным ключом
bedrock-pack-tools decrypt <pack-dir> <key> [output-dir]

# Пакетный режим с использованием keys.json из `keys` или `download`
bedrock-pack-tools decrypt --all <keys.json> <packs-dir> [output-dir]
```

В пакетном режиме зашифрованные паки сопоставляются с ключами по UUID в
`manifest.json` каждого пака, поэтому структура каталогов не важна. Ключ - это
32-символьный мастер-ключ AES; пофайловые ключи находятся внутри зашифрованного
`contents.json`.

### `encrypt` - шифрование ресурс-пака в `.mcpack`

```bash
bedrock-pack-tools encrypt [--key-out PATH] <pack-dir> [key] [output.mcpack]
```

Упаковывает обычный каталог в `.mcpack` плюс `.mcpack.key` рядом с ним. Каждый
файл, кроме манифеста, получает собственный случайно сгенерированный 32-символьный
ключ AES-256-CFB8; `manifest.json` и `pack_icon.png` копируются как есть
(перечислены в `contents.json` с пустым ключом, как ожидает Bedrock).

Если ключ не указан, генерируется 32-символьный буквенно-цифровой ключ. Если не
указан путь вывода, `.mcpack` получает имя каталога.

`--key-out PATH` (`-k PATH`) записывает мастер-ключ по выбранному вами пути вместо
стандартного `<output.mcpack>.key` рядом с паком - удобно, чтобы не держать ключи
в каталоге, который вы передаёте.

Для развёртывания: положите `.mcpack` и `.mcpack.key` в каталог `resource_packs/`
вашего сервера Bedrock и зарегистрируйте пак так, как обычно делает ваш сервер.

### `featured` - просмотр Featured Servers

```bash
bedrock-pack-tools featured                            # вывести всё
bedrock-pack-tools featured download <index> [output-dir]
```

Выводит каталог Featured Servers от Mojang (тот, что использует вкладка Servers в
игре) плюс любые активные Live Events, с живым RakNet-пингом для каждой записи.
Имена хостов берутся прямо из этого API, поэтому список остаётся актуальным по мере
того, как партнёры переезжают.

В каждой строке пять колонок - метка, индекс, имя, адрес, статус:

```
  [ON]   [ N]  <server name>         <host>:<port>            online <count> players
  [OFF]  [ N]  <server name>         <host>:<port>            offline
  [EXP]  [ N]  <server name>         (experience-join)        resolve on download
  [EVT]  [ N]  <event name>          (live event)             resolve on download
```

- `[ON]`  - партнёр с публичным `host:port`, в сети (RakNet-пинг прошёл)
- `[OFF]` - партнёр с публичным `host:port`, не в сети (пинг не прошёл)
- `[EXP]` - доступен через `experienceId`, без фиксированного хоста; разрешается при скачивании через `POST /api/v2.0/join/experience`
- `[EVT]` - Mojang Gathering / live event; разрешается при скачивании через `/api/v1.0/access` + `/api/v1.0/venue/{gatheringId}`

### `login` - вход через Xbox Live

```bash
bedrock-pack-tools login
```

Запускает поток device code Xbox Live, если токен не кэширован, затем кэширует
его. Используйте, чтобы пройти авторизацию заранее, до запуска `keys`, `download`
или `featured` (или интерактивного меню), вместо того чтобы получать запрос
посреди действия.

### `logout` - очистка кэшированных токенов

```bash
bedrock-pack-tools logout
```

Удаляет кэшированный токен Xbox и MCToken, так что следующая команда с авторизацией
снова запросит вход. Когорта `.device_id` остаётся на месте.

### `version` - вывод версии сборки

```bash
bedrock-pack-tools version   # также -v / --version
```

Выводит тег, проставленный goreleaser для выпущенного бинарника, или версию модуля
(через `debug.ReadBuildInfo`) при сборке из исходников.

## Форматы файлов

### `keys.json`

Карта UUID пака → информация о шифровании, записывается командами `keys` /
`download` и используется `decrypt --all`:

```json
{
  "12345678-1234-1234-1234-123456789abc": {
    "key": "ABCDEFGHIJKLMNOPQRSTUVWXYZ123456",
    "version": "1.0.0",
    "name": "Example Pack"
  },
  "fedcba98-7654-3210-fedc-ba9876543210": {
    "key": "ZYXWVUTSRQPONMLKJIHGFEDCBA987654",
    "version": "2.3.1",
    "name": "Another Pack"
  }
}
```

### Структура зашифрованного пака

Файл `contents.json` внутри зашифрованного пака - это 256-байтовый бинарный
заголовок, за которым следует шифротекст AES-256-CFB8.

| Смещение  | Байты | Содержимое                                          |
| --------- | ----- | --------------------------------------------------- |
| `0..3`    | 4     | Версия (uint32, little-endian, сейчас `0`)          |
| `4..7`    | 4     | Магические байты `0xFC 0xB9 0xCF 0x9B`              |
| `8..15`   | 8     | Нулевое заполнение                                  |
| `16`      | 1     | Разделитель `'$'` (0x24)                            |
| `17..52`  | 36    | UUID пака, ASCII                                    |
| `53..255` | 203   | Нулевое заполнение                                  |

После заголовка идёт JSON-документ, зашифрованный мастер-ключом (из handshake
сервера или файла `.mcpack.key`). IV - это первые 16 байт 32-байтового ключа.

Расшифрованный JSON перечисляет каждый файл в паке с его собственным пофайловым
ключом. `manifest.json` и `pack_icon.png` имеют пустые ключи и хранятся
незашифрованными.

## FAQ

**Как расшифровать ресурс-пак Minecraft Bedrock?**
Если у вас уже есть 32-символьный ключ, выполните `decrypt <pack-dir> <key>`. Если
у вас есть `keys.json` из `keys` или `download`, выполните `decrypt --all <keys.json>
<packs-dir>`, чтобы расшифровать все паки сразу - сопоставление паков с ключами идёт
по UUID в каждом `manifest.json`. Расшифровывайте только те паки, которыми владеете
или которые вам разрешено проверять.

**Как получить ключ шифрования ресурс-пака Bedrock?**
Команда `keys` входит через Xbox Live и снимает AES-ключи контента прямо с сервера,
которым вы управляете или который вам разрешено проверять, так что ключ заранее не
нужен. На выходе - карта UUID-к-ключу, которую можно передать в `decrypt --all`.

**Как скачать все ресурс-паки с сервера Bedrock?**
Выполните `download <server:port>`, чтобы скачать все паки, которые отдаёт сервер,
плюс файл ключей одной командой. Добавьте `--decrypt`, чтобы получить готовые к
использованию расшифрованные папки за тот же шаг.

**Как преобразовать `.mcpack` в `.zip`?**
`.mcpack` - это zip-архив, переименуйте расширение в `.zip` и распакуйте. Если
содержимое зашифровано (бинарный заголовок `contents.json`), сначала выполните
`decrypt` с подходящим ключом, чтобы получить обычные файлы.

**Что такое 32-символьный ключ и где находятся пофайловые ключи?**
Это мастер-ключ AES-256 пака. Пофайловые ключи находятся внутри зашифрованного
`contents.json` пака; инструмент читает их автоматически, как только получит
мастер-ключ. IV - это первые 16 байт 32-байтового ключа.

**Можно ли заново зашифровать собственный пак в `.mcpack`?**
Да. `encrypt <pack-dir>` упаковывает обычный каталог в `.mcpack` плюс `.mcpack.key`,
генерируя свежий 32-символьный ключ AES-256-CFB8 на каждый файл. Положите оба файла
в каталог `resource_packs/` вашего сервера для развёртывания.

**Это разрешено?**
Инструмент создан для исследователей безопасности, операторов серверов,
проверяющих собственные развёртывания, и авторов паков, восстанавливающих свои
ключи. Не используйте его для распространения чужого платного контента.

## Устранение неполадок

**`featured` не показывает серверов или показывает другие серверы, чем в игре.**
Mojang делит каталог по когорте PlayFab Experiments, привязанной к `device.id`.
Инструмент сохраняет стабильный ID в `.device_id`, чтобы ваш список оставался
неизменным между запусками - и он может отличаться от вашего игрового клиента,
если вы попали в другую когорту. Используйте **Settings > Reset featured cohort**
в интерактивном меню (или удалите `.device_id` и перезапустите), чтобы получить
новую когорту.

**Строка `[EVT]` падает при скачивании.**
Live events появляются и исчезают. Если для gathering в момент попытки
подключения нет активного venue, API ничего не возвращает, и инструмент выводит
понятное сообщение. Подождите или выберите другую запись.

**Handshake зависает на конкретном сервере.**
Инструменту нужен пропатченный форк
[gophertunnel](https://github.com/iteplenky/gophertunnel/tree/fix/deferred-packet-race),
зафиксированный в `go.mod` (строка require `github.com/iteplenky/gophertunnel`), чтобы
исправить гонку отложенных пакетов, которая приводит к взаимной блокировке на
некоторых серверах. Если вы собрали из исходников и вернули этот require обратно на
основной gophertunnel - вот в чём причина.

**Расшифровка выдаёт мусор.**
Неверный ключ или зашифрованный пак был скачан не полностью (например, обрезанный
ответ CDN). Перезапустите `download -v`, чтобы убедиться, что передача пака
завершилась.

**Авторизация постоянно снова запрашивает device code.**
Кэш токена находится в `os.UserConfigDir()` (см. [Вход через Xbox Live](#вход-через-xbox-live)
для пути на каждой ОС). Если каталог недоступен для записи - например, при запуске
в песочнице - каждый запуск начинает авторизацию заново.

## Переменные окружения

| Переменная         | Эффект                                                                          |
| ------------------ | ------------------------------------------------------------------------------- |
| `NO_COLOR`         | Отключает все ANSI-цвета и escape-коды ([no-color.org](https://no-color.org/))   |
| `BPT_DIAL_TIMEOUT` | Переопределяет таймаут подключения для `keys` / `download`, как Go duration (`5m`, `90s`). Полезно для медленных серверов или долгого ожидания информации о паках. Ограничивает весь запуск keys / download - начальное подключение и любые CDN-загрузки делят этот дедлайн. |

## Заметки

- `.xbox_token.json` и `.mctoken.json` - это полноценные токены авторизации, не
  делитесь ими.
- Ключи специфичны для сервера. Разные серверы используют разные ключи.
- Паки, которые сервер помечает как незашифрованные, перечисляются, но
  исключаются из `keys.json`.
